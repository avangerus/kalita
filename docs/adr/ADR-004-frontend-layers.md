# ADR-004: Три слоя фронта — нотация, рендерер, продукт (модель WordPress)

**Status:** Accepted (2026-06-14). Закрепляет формулировку фаундера: «бек имеет
нотации, фронт собирает по ним интерфейс, поверх — сайт как в WordPress».
Уточняет ADR-003 и снимает боль «React внутри бинаря — геморрой».

## Три слоя

1. **Бек — источник истины и НОТАЦИЯ.** `/api/meta` диктует, что существует:
   сущности, поля, типы, доступные актёру действия (кнопки), фильтры, доски,
   возможности узла (search и т.п.). Бек не обслуживает фронт — он его
   определяет. Ноль презентационных решений в ядре.

2. **Фронт — чистый РЕНДЕРЕР нотации.** Не содержит доменной логики (не знает
   про дебиторку/knowvault), берёт meta и рисует. Любой рендерер
   взаимозаменяем: встроенный preact-клиент, приложение на @kalita/sdk, чужой
   фронт — все потребляют одну нотацию.

3. **Продукт/сайт — поверх рендерера (как WordPress).**
   WP core = данные+API ≈ наш бек+meta. WP-темы = рендерер ≈ наш SDK/клиент.
   Конкретный сайт (магазин, KnowVault, кабинет с оплатой) = тема + контент.
   Конечный пользователь видит сайт, не зная ни про темы, ни про ядро (T12).

## Следствие для упаковки: фронт развязан с бинарём

Геморрой = когда фронт зашит в бинарь и каждая правка требует пересборки Go
(и ловит баги «отдаёт старый JS»). Решение:

- **dev/кастом:** `serve --ui-dir ./web` — статика с диска. Правишь файл → F5,
  Go не трогаешь. Свой сайт/тему кладёшь сюда.
- **коробка:** встроенный рендерер остаётся за build-тегом `embedui` —
  `go build -tags embedui` даёт одну самодостаточную коробку для on-prem.
- по умолчанию (без тега и без --ui-dir): узел отдаёт только API — чистый
  бекенд, фронт подключается снаружи (SDK).

preact+htm у встроенного рендерера — НЕ сборка (один .js, кладётся как есть);
«геморрой React» относился к бандлерам (webpack/vite + embed готового бандла),
куда проект сознательно НЕ идёт.

## Правило

Любая презентация — слой 2/3, никогда не ядро. Новый рендерер не получает
прав мимо API. «Тема» (сайт) не трогает данные мимо meta-прав (граница
порядок/свобода из PORTAL-VISION).

## Alternatives
(1) React+бандлер в бинаре — отвергнуто (геморрой пересборки, против коробки).
(2) Только встроенный фронт — отвергнуто (тупик для тем/партнёров).
(3) Фронт всегда снаружи — отвергнуто как дефолт: ломает «один docker run»
для on-prem админки; поэтому embed остаётся опцией за тегом.
